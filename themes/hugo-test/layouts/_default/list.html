Теперь необходимо поработать над шаблонами, чтобы добавить функциональности и реализовать вывод различного контента:

  1. в футере сайта в копирайте вывести текущий год; вывести название компании заданное в параметрах сайта (параметр siteCopyright)
  2. в меню убрать ссылку с пункта Главная, если открыта главная страница
  3. в остальных пунктах меню также сделать, чтобы на пункте меню текущей открытой страницы не было ссылки и он был выделен жирным шрифтом
  4. вывести второй уровень меню для пунктов, содержащих вложенные пункты (с учётом требований выше)
  5. вывести на корневую страницу каталога список подстраниц со ссылками, первый пункт списка выделить жирным шрифтом
  6. добавить шаблоны для вывода следующих видов контента:
    • список всех моделей с сортировкой по полю position
    • список автомобилей в наличии бренда Mazda
    • список автомобилей в наличии бренда Renault
    • список автомобилей 2024 года выпуска
    • список моделей, автомобили которых есть в наличии
  7. добавить вывод на все странцы хлебных крошек

Ниже представлена информация о том, как должны называться шаблоны для каждого вида контента из п.6, а также дополнительная поясняющая информация по каждому из них.

Все шаблоны необходимо размещать в директории `themes/hugo-test/layouts/partials/content`. В этой директории уже есть шаблон для списка брендов — `brands.html`. Как выглядит результат интеграции данных о брендах в шаблон `brands.html`, можно посмотреть здесь: http://localhost:1313/catalog/brands/. Для удобного доступа к странцам с контентом нужно выполнить пункты 4 и/или 5.

Все шаблоны должны выводить таблицу элементов. Структура таблицы для каждого вида контента будет описана ниже.

Список всех моделей с сортировкой по полю position

Имя файла шаблона: `models.html`

Структура контента:

    Название модели,
    Название бренда,
    Идентификатор модели из поля id,
    Порядковый номер из поля position

Пояснение
Фомирование данного списка схоже с формированием списка брендов. Только необходимо добавить сортировку объектов по значению поля position. Как это сделать — ответ в документации hugo.

Список автомобилей в наличии бренда Mazda

Имя файла шаблона: `vehicles-mazda.html`

Структура контента:

    [Название бренда] [Название модели],
    Цвет модели,
    Год выпуска,
    Цена

Пояснение
При формировании данного списка необходимо включать только те записи автомобилей (vehicles), у которых указан бренд Mazda.
Ещё необходимо обратить внимание на то, что имя автомобиля формируется из названия бренда и названия модели.

Полезная информация

Документация Hugo по работе с датой и временем: https://gohugo.io/functions/time/

Документация Hugo по созданию шаблонов для вывода меню: https://gohugo.io/templates/menu/, https://gohugo.io/methods/menu-entry/

Для решения потребуется информация о переменных: как их инициализировать, как менять значение.

Чтобы получить данные, хранящиеся в директории `data`, необходимо использовать `partial`: partial "utils/get-data" "brands". См. шаблон brands.html

Вложенные структуры данных предоставляют доступ ко вложенным уравням через оператор точки. Например, data.name

Также стоит обратить внимание на конструкцию "точка" — . Это название переменной, которая содержит текущий контекст. На уровне шаблона точка содержит все доступные шаблону данные. Некоторые операторы, такие как `range`, меняют содержимое контекста внутри своего блока. Поэтому точка содержит другое значение внутри блока `range` и ему подобным, которое определяется логикой блока.

Для целей тестирования можно выводить значение переменных в любом месте шаблона, чтобы посмотреть, что в них содержится.
